mojio / mojizo 取扱説明書

mojioはスマホアプリ、mojizoは録音デバイス、mozzyはこの開発リポジトリです。録音、SDカードからの吸出し、Google Driveアップロード、ゲイン調整、スマホからのGitHubビルドまでをこの1枚にまとめています。

アプリmojioAndroidで接続・吸出し・アップロード
デバイスmojizoBLE録音デバイス / XIAO nRF52840 Sense
リポジトリmozzyアプリ・ファーム・補助ツール一式

最初に準備するもの

  • mojizo本体、microSDカード、USBケーブル。
  • Androidスマホ。Bluetooth、位置情報、通知、バックグラウンド動作の許可が必要です。
  • Google DriveにアップロードするGoogleアカウント。
  • 開発や再ビルドをする場合はGitHubアカウントと、リポジトリ shkentee/mozzy へのアクセス。
表示名のルール: アプリ名は mojio、BLEデバイス名は mojizo、GitHubリポジトリ名は mozzy です。古い検証用ファームが入っている場合だけ、BLE上では Friend と見えることがあります。

基本の流れ

  1. mojizoを起動します。
    microSDカードを入れ、電源を入れます。スマホのBluetoothをオンにします。
  2. mojioを開きます。
    初回はスキャン画面で mojizo を選びます。旧検証ファームでは Friend と表示される場合があります。
  3. 録音を開始します。
    アプリの SDに録音 スイッチ、またはデバイスの録音操作でSDカード録音をオンにします。
  4. 吸出しを実行します。
    自動設定の場合、mojioがmojizoのSDカード上の録音データをスマホへ吸い出し、5分単位のチャンクとしてアップロード待ちにします。手動設定の場合は吸出しボタンを押します。
  5. Google Driveへアップロードします。
    自動アップロードがオンなら、通信条件を満たすとアップロードされます。手動設定の場合はアップロード操作を行います。

録音

アプリから録音する

  1. mojioでmojizoに接続します。
  2. デバイス画面の SDに録音 をオンにします。
  3. 録音を止めるときは同じスイッチをオフにします。

デバイス側で録音する

mojizo側の録音操作でもSDカード録音を開始・停止できます。録音中はSDカードへOpus形式の音声データが保存されます。

録音中に電源を抜くと、最後の短い区間が保存されないことがあります。停止操作のあと数秒待ってから電源を切るのが安全です。

吸出し

mojioはmojizoのSDカードから録音ファイルをBLE経由で取り込みます。新しいmojizoファームでは範囲指定で安定して吸い出し、旧検証ファームでは旧方式の全体吸出しに自動で切り替えます。

  • 自動: スケジュールや間隔に応じて吸出しを行います。
  • 手動: 必要なときだけ吸出しボタンで実行します。
  • 保存単位: 取り込んだ音声はDriveアップロードに向くよう、一定時間ごとのファイルに分けられます。

Google Driveアップロード

  1. 設定画面でGoogleアカウントにログインします。
  2. アップロード先フォルダを確認します。既定の検証フォルダIDは 1IPNXw8EzMz6u6nGUo5H1xtuwkI4NKayJ です。
  3. 自動アップロードをオンにすると、吸い出した録音が順番にアップロードされます。
  4. モバイル通信を避けたい場合は Wi-Fiのみ をオンにします。
同じファイル名・同じサイズのファイルは重複アップロードしないように扱います。アップロード後、スマホ内の送信待ちファイルは削除されます。

ゲイン調整

対応ファームウェアでは、mojioからマイク入力ゲインを変更できます。表示はOMI互換のdB表記で、Mute、-20dB、-10dB、+0dB、+6dB、+10dB、+20dB、+30dB、+40dB の9段階です。

状況動作
新しいmojizoファームOMI互換の設定サービスで0から8のゲイン段階を公開し、OMI公式と同じPDMゲイン値へ変換します。
診断用raw gainサービス開発・検証用にPDMゲイン値を直接読む特性も残します。
ゲイン特性がないファームゲイン操作は表示されません。録音・吸出し・アップロードはそのまま使えます。
mojizoファームはOMI互換の 19b10010 / 19b10012 でゲイン0..8を公開します。初期値はレベル6（+20dB）で、内部PDMゲインはOMI DevKit2と同じ MIC_GAIN 64 です。高いゲインはノイズやクリップも増えやすいため、録音環境に合わせて調整してください。

スマホからGitHubでビルドする

スマホに開発環境を入れなくても、GitHub Mobileまたはスマホブラウザからビルドできます。

APKを作る

  1. https://github.com/shkentee/mozzy/actions を開きます。
  2. mobile ワークフローを選び、Run workflow を押します。
  3. 完了した実行結果から mozzy-release-<sha> をダウンロードします。
  4. ZIPを展開し、app-release.apk をAndroidへインストールします。

UF2ファームを作る

  1. firmware ワークフローを選び、Run workflow を押します。
  2. 完了した実行結果から mozzy-firmware-xiao-ble-<sha> をダウンロードします。
  3. mojizoをブートローダーモードにして、表示されたドライブへ firmware.uf2 をコピーします。

検証メモ

  • BLE接続、バッテリー表示、SDファイル一覧、PC補助ツールでの吸出しとWAV復元は実機で確認済みです。
  • 実機で確認した吸出し例: 1780826727.opus_sd、127,836 bytes、復元WAVは約29.76秒。
  • アプリ側の旧ファーム吸出し互換を追加し、0バイトで同期完了扱いにならないよう修正済みです。
  • mojio上で0.2MBの吸出し完了、未アップロードなし、アップロード済みIDの記録を確認済みです。
  • このCodexセッションのGoogle Driveコネクタでは対象フォルダが空に見えました。mojioにログインしているGoogleアカウントと、確認に使うDriveアカウントが一致しているか確認してください。
  • ゲインはOMI互換のdB表記とPDMゲイン値に合わせて実装しています。実機確認結果はREADMEと引き継ぎファイルに追記します。
  • GitHub Actionsの mobile、firmware、tools はpush後に成功を確認済みです。スマホからも同じActions画面で手動実行できます。
